## Confirmed safety boundaries

The app does not currently perform real production actions.

The following are intentionally not connected yet:

- Real authentication
- Real user sessions
- Real company account data
- Real booking submission
- Real contact request submission
- Real email sending
- Real payment or subscription handling
- Backend API integration
- Database writes
- Push notifications

Booking and contact flows are shown as prototype/demo flows only.

## Recent prototype polish

Recent completed polish phases improved:

- Service detail copy
- Pricing CTA and payment safety copy
- Contact prototype copy
- Account and settings demo copy
- Confirmation and legal/prototype copy

The app now avoids implying that real requests, bookings, payments, or account changes are active.

## Current product role

This app is currently suitable for:

- Internal demo
- Product walkthrough
- Visual prototype
- Planning the real mobile experience
- Aligning future app, website, and backend work

It is not yet ready for production customer use because backend, auth, booking submission, and payment are not connected.

## Next recommended phases

1. Sync app copy with the Proffera website when the website is closer to final.
2. Define backend/API requirements before enabling real booking submission.
3. Add authentication only after the account model and backend are ready.
4. Add real contact/booking flows only with validation, error handling, and dashboard integration.
5. Add app icon, splash screen, privacy policy, terms, and store metadata before public release.

## Release note

Do not enable real booking, payment, email, auth, or backend behavior without a separate reviewed phase.
